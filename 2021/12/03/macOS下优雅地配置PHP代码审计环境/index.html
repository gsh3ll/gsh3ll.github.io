最近在学习 PHP 代码审计，在虚拟机用 phpstudy+seay 搭建调试环境，但是在虚拟机中总感觉不是很顺手，网上搜资料时发现了国光 macOS 下优雅地配置 PHP 代码审计环境，照搬过来发现用起来很顺手，颜值也高，遂记录。

# Tools

  • macOS 11.6：bigsur 系统
  • MAMP PRO 6.6：PHP 集成环境
  • Navivat Premium：数据库图形化管理工具
  • Xdebug：PHP 动态调试必备扩展

# MAMP

下载地址：MAMP

默认安装即可。

面板的 Port&User 标签页可以看到一些基本的环境信息：

组件端口
Apache8888
Nginx7888
MySQL8889
默认 Web 根目录/Users/User_name/Sites/localhost

# MAMP 之 Mysql 配置

MySQL 配置主要参考下面的截图：

  1. Change password 这里重置 MySQL 的 root 密码很方便，服务停止状态下可以直接修改 root 用户密码。
  2. 勾选上之后，可以本地使用类似于 Navicat 的软件来连接本地的 MySQL
  3. MAMP 也自带 PHPMyAdmin 这种 Web 端的 MySQL 管理工具，PHP >= 5.5 环境下即可正常激活使用。

# MAMP 之 PHP 配置

PHP 配置主要参考下面的截图：

  1. 切换 PHP 版本，支持：5.4.45、5.6.40、7.0.33、7.1.32、7.2.22、7.3.9、7.4.2
  2. Open template，点击可以直接打开对应 PHP 版本的 ini 配置文件
  3. 开启 Xdebug 扩展，是一个好用的 PHP 扩展，提供调试和性能分析功能，代码审计必备

# 新建网站

接下来尝试使用 MAMP 来安装一下这个有漏洞的 CMS。配置细节直接参考下图：

下面来简单讲解一下上面的各个标记的含义：

  1. 添加一个新的 hosts 站点
  2. 添加站点的名称
  3. 自定义站点的名称，此时国光我这里的命名为 x'x ，那么此时这个站点就可以通过如下 URL 进行访问了：http://xxx:8888
  4. 点击可以直接在浏览器打开这个站点
  5. 自定义站点网站根目录的路径信息
  6. 实际上访问 xxx 可以打开刚刚网站的原理就是因为 MAMP 编辑了我们本地的这个 hosts 文件了

接着就是常规 CMS 安装了，填写数据库用户名和密码，然后设置网站后台密码等。

# 编辑器 / IDE

# vscode

需要安装的插件：

  • chinese（翻译）
  • indent-rainbow（高亮缩进）
  • Bracket Pair Colorizer（括号高亮）
  • Better Comments（注释高亮）
  • vscode-icons-mac（文件图标主题）
  • PHP Debug（用作 php 代码审计）
  • PHP Intelephense（辅助代码审计）
