Hexo 是一个基于 node.js 开发，开源的静态博客生成器。此文章用作记录搭建该博客的过程，以及中途遇到的问题。

# 搭建准备

Hexo 是一个开源的静态博客生成器，用 node.js 开发，所以在安装 Hexo 前我们需要先安装 node.js。

卸载的话：

sudo rm -rf /usr/local/{bin/{node,npm},lib/node_modules/npm,lib/node,share/man/*/node.*}

进入 node.js 主页后下载 LTS 版本，双击安装即可。安装完成后可通过下面命令查看版本信息。

gsh3ll:~ $ node -v
v14.17.3
gsh3ll:~ $ npm -v 
6.14.13

通过 npm 来安装 Hexo，但是受国内环境影响，下载速度很慢。可通过安装 cnpm 来解决。

gsh3ll:~ $ sudo npm install -g cnpm --registry=https://registry.npm.taobao.org
gsh3ll:~ $ cnpm -v	##查看cnpm版本
cnpm@7.0.0 (/usr/local/lib/node_modules/cnpm/lib/parse_argv.js)
npm@6.14.14 (/usr/local/lib/node_modules/cnpm/node_modules/npm/lib/npm.js)
node@14.17.3 (/usr/local/bin/node)
npminstall@5.0.1 (/usr/local/lib/node_modules/cnpm/node_modules/npminstall/lib/index.js)
prefix=/usr/local 
darwin x64 20.5.0 
registry=https://registry.nlark.com

最后使用 cnpm 安装 Hexo 即可。

gsh3ll:~ $ sudo cnpm install -g hexo-cli
gsh3ll:~ $ hexo -v    
hexo-cli: 4.3.0
...
unicode: 13.0

# 搭建网站

在本地空间中找个合适的地方新建文件夹，用来存放 Hexo 的文件，并在该文件夹路径下初始化博客。

gsh3ll:MyBlog $ pwd
/Users/gsh3ll/MyBlog
gsh3ll:MyBlog $ npm config set registry https://registry.npm.taobao.org		## 切换npm为淘宝源
gsh3ll:MyBlog $ sudo hexo init 	# 初始化Hexo
INFO  Cloning hexo-starter https://github.com/hexojs/hexo-starter.git
Cloning into '/Users/gsh3ll/Blog'...
...
INFO  Start blogging with Hexo!

至此博客已经初步搭建成功了，只需要使用命令 hexo start 或 hexo s 便会在本地的 4000 端口开启一个 web 服务。

浏览器访问 http://localhost:4000，就能打开博客了。

博客已经搭建完成了，如何写属于自己的文章呢？

使用命令 hexo new [layout] <title> 来创建第一篇文章。

gsh3ll:Blog $ sudo hexo n first      ##创建文章
INFO  Created: ~/Blog/source/_posts/first.md        ##文章默认存放在/source/_posts/目录下

随后对生成的 md 文件进行编辑，写入内容。

gsh3ll:Blog $ hexo clean      ##清除hexo
INFO  Deleted database.
gsh3ll:Blog $ hexo generate    ##生成页面，亦可使用命令hexo g
INFO  Start processing
INFO  Files loaded in 123 ms
...
INFO  29 files generated in 364 ms

# 远端部署

下面需要把本地的博客部署到网络上的 github 上。

首先我们需要登录到 github，并新建一个仓库，用户部署个人博客的 github 仓库的命名必须要符合特定的要求，即仓库的名称必须是 你的昵称 。

并且复制 github 仓库的地址，以备后续需要。

新建完 github 仓库后，我们需要在本地博客文件夹路径下安装一个 git 插件 hexo-deployer-git。

gsh3ll:Blog $ pwd
/Users/gsh3ll/Blog
gsh3ll:Blog $cnpm install --save hexo-deployer-git ##或npm install hexo-deployer-git --save
✔ Installed 1 packages
✔ Linked 0 latest versions
✔ Run 0 scripts
✔ All packages installed (used 59ms(network 58ms), speed 24.43kB/s, json 1(1.42kB), tarball 0B)

随后我们需要对 _config.yml 这个文件进行一些配置。

##在_config这个文件的最后，修改如下，需要注意每个冒号后面要有空格
# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: 你的github仓库URL
  branch: master

保存配置文件后便能够往 github 上推我们的本地文件。

gsh3ll:Blog $ hexo d                                         
INFO  Deploying: git
...
 create mode 100644 js/script.js
Username for 'https://github.com': xxxx       ##这里会提示你输入github的账号
Password for 'https://iamg7s@github.com':       ##这里会提示你输入github的密码
...
INFO  Deploy done: git
root@missing Blog #

随后查看 github 仓库，发现本地文件已经被 push 到 github 了。

这时你便可以通过访问你的仓库名称来访问你的博客了，比如我的博客地址为： https://gsh3ll.github.io/ 。

# 更换主题

默认的主题有点丑陋，我们可以在这里 https://hexo.io/themes/ 找到自己喜欢的主题并应用。

这里我选择的是 Shoka 主题。

进入到本地博客的目录下，使用 git 命令拉取该主题，并把下载的主题存放到 themes 目录下。随后返回博客目录，修改 _config.yml 文件。

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: landscape        ##修改成你下载的主题的名字，这里landscape是Hexo默认的主题名字
# Deployment

打开主题目录，内有 example 文件夹，将里面的 _config.yml 复制替换根目录 _config.yml 这样少了一步配置搭建为了节省时间。

在应用主题之前，必须至少安装 hexo-renderer-multi-markdown-it 渲染插件 和 hexo-autoprefixer 。

安装前，记得务必卸载掉默认的 hexo-renderer-marked ，以及别的 markdown 文件渲染器

npm un hexo-renderer-marked --save
or
npm i hexo-renderer-multi-markdown-it --save --ignore-scripts

安装完成后就可以生成预览网站了：

hexo clean
hexo g -d

如果网站报错，则修改一下版本即可：

npm uninstall js-yaml
npm install js-yaml@3.14.1

# 修改主题配置

主题配置的所有参数在 /themes/shoka/_config.yml 文件中。 为了方便主题升级，请在根目录新建一个 yml 文件，命名为 _config.shoka.yml 。 也就是说，所有主题的自定义配置均保存于 /_config.shoka.yml 文件。
