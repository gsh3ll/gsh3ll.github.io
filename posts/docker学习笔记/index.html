Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中，然后发布到任何流行的 Linux或Windows机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

0x00 Docker的简介#

Docker的简介什么的就不详细介绍了，优点如下：更高效的利用系统资源、更快速的启动时间、一致的运行环境、持续交付和部署、更轻松的迁移、更轻松的维护和扩展。

0x01 Docker的安装#

mac#

在官网注册下载软件后，直接拖入application后即可。

linux#

安装https协议、CA证书、dirmngr

apt-get update
apt-get install -y apt-transport-https ca-certificates
apt-get install dirmngr

添加GPG密钥并添加更新源

apt-key adv --keyserver hkp://p80.pool.sks-keyservers.net:80 --recv-keys 58118E89F3A912897C070ADBF76221572C52609D 
echo 'deb https://apt.dockerproject.org/repo debian-stretch main' &gt; /etc/apt/sources.list.d/docker.list

系统更新以及安装docker

apt-get update
apt-get install docker-engine

启动docker服务器

service docker start

安装compose

pip install docker-compose

0x02 Docker的基本命令#

1.镜像的搜索。

docker search [镜像名]

2.docker镜像拉取下载。

docker pull [镜像名称]

下载镜像的速度会特别慢，可以在docker创建或修改 /etc/docker/daemon.json 文件，来设置里面添加国内的镜像加速器。

{
  "registry-mirrors" : [
    "http://ovfftd6p.mirror.aliyuncs.com",
    "http://registry.docker-cn.com",
    "http://docker.mirrors.ustc.edu.cn",
    "http://hub-mirror.c.163.com"
  ],
  "insecure-registries" : [
    "registry.docker-cn.com",
    "docker.mirrors.ustc.edu.cn"
  ],
  "debug" : true,
  "experimental" : true
}

3.创建并连接容器、端口映射、绑定随机id。

docker run -i -t -p 8081:80  -p 8080:8080  -p 4444:4444  -p 5555:5555  -p 2222:22  kalilinux/kali-linux-docker  /bin/bash

下面的是常用的命令：

docker images                     //列出下载的镜像
docker run -i -t --name hellodocker ubuntu /bin/bash   
//创建并使用容器;-i、-t参数可以在运行的命令行中进行输入与输出;使用—-name来自定容器的名称。该参数未赋值，docker默认会自动生成随机的名称；另外-d参数表示在后台运行。
exit                             //退出容器
docker ps -a                     //查看容器列表
docker start [容器名]             //启动容器
docker restart [容器名]           //重启容器
docker stop [容器名]              //终止容器
docker rm [容器名]                //删除容器
docker rmi [镜像名]               //删除镜像,需要注意的是，要先停止、删除应用该镜像的容器
docker attach [容器名]            //start一个容器后，默认是不进入容器的，需要我们连接容器
docker stats                     //显示一个或多个容器的统计信息，可以看到容器的 ID、CPU 占用率、内存使用率、网速等信息;

0x03 其他一些命令#

docker save [镜像名] -o xxxx.tar  //导出镜像
docker load -i xxxx.tar          //导入镜像
docker tag [IMAGE ID] xxx:xxx    //修改tag名称
//默认情况下，在容器中输入exit会退出容器并停止，我们使用exec参数即可实现输入exit后依旧在后运行。
