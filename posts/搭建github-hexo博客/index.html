Hexo是一个基于node.js开发，开源的静态博客生成器。此文章用作记录搭建该博客的过程，以及中途遇到的问题。因为已经弃用七牛云的缘故，本文的图片已经无法显示，故图片已被删除。

软件安装#

Hexo是一个开源的静态博客生成器，用node.js开发，所以在安装Hexo前我们需要先安装node.js。

node.js下载地址：https://nodejs.org/zh-cn/

进入node.js主页后下载LTS版本，双击安装即可。安装完成后可通过下面命令查看版本信息。


g7s:~ $ node -v     ##查看node版本
v12.15.0
g7s:~ $ npm -v      ##查看npm版本
6.13.4

通过npm来安装Hexo，但是受国内环境影响，下载速度很慢。可通过安装cnpm来解决。


g7s:~ $ npm install -g cnpm --registry=https://registry.npm.taobao.org
g7s:~ $ cnpm -v     ##查看cnpm版本

开始搭建#

在本地空间中找个合适的地方新建文件夹，用来存放Hexo的文件。在该文件夹路径下初始化博客。


g7s:Blog $ pwd
/Users/g7s/Blog
g7s:Blog $ sudo hexo init       ##初始化博客
INFO  Cloning hexo-starter https://github.com/hexojs/hexo-starter.git
Cloning into '/Users/g7s/Blog'...
...
INFO  Start blogging with Hexo!

至此博客已经初步搭建成功了，只需要使用命令hexo start 或 hexo s 便会在本地的4000端口开启一个web服务。

浏览器访问http://localhost:4000，就能打开博客了。

博客已经搭建完成了，如何写属于自己的文章呢？

使用命令hexo new [layout] <title> 来创建第一篇文章。


g7s:Blog $ sudo hexo n first      ##创建文章
INFO  Created: ~/Blog/source/_posts/first.md        ##文章默认存放在/source/_posts/目录下

随后对生成的md文件进行编辑，写入内容。


g7s:Blog $ hexo clean      ##清除hexo
INFO  Deleted database.
g7s:Blog $ hexo generate    ##生成页面，亦可使用命令hexo g
INFO  Start processing
INFO  Files loaded in 123 ms
...
INFO  29 files generated in 364 ms

远端部署#

下面需要把本地的博客部署到网络上的github上。

首先我们需要登录到github，并新建一个仓库，用户部署个人博客的github仓库的命名必须要符合特定的要求，即仓库的名称必须是你的昵称。

并且复制github仓库的地址，以备后续需要。

新建完github仓库后，我们需要在本地博客文件夹路径下安装一个git插件hexo-deployer-git。


g7s:Blog $ pwd
/Users/g7s/Blog
g7s:Blog $cnpm install --save hexo-deployer-git ##或npm install hexo-deployer-git --save
✔ Installed 1 packages
✔ Linked 0 latest versions
✔ Run 0 scripts
✔ All packages installed (used 59ms(network 58ms), speed 24.43kB/s, json 1(1.42kB), tarball 0B)

随后我们需要对_config.yml这个文件进行一些配置。


##在_config这个文件的最后，修改如下，需要注意每个冒号后面要有空格
# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: 你的github仓库URL
  branch: master

保存配置文件后便能够往github上推我们的本地文件。


g7s:Blog $ hexo d                                         
INFO  Deploying: git
...
 create mode 100644 js/script.js
Username for 'https://github.com': iamg7s       ##这里会提示你输入github的账号
Password for 'https://iamg7s@github.com':       ##这里会提示你输入github的密码
...
INFO  Deploy done: git
root@missing Blog #

随后查看github仓库，发现本地文件已经被push到github了。

这时你便可以通过访问你的仓库名称来访问你的博客了，比如我的博客地址为：https://iamg7s.github.io/。

更换主题#

默认的主题 有点丑陋QAQ，我们可以在这里https://hexo.io/themes/找到自己喜欢的主题并应用。

这里我选择的是fluid主题。

进入到本地博客的目录下，使用git命令拉取该主题，并把下载的主题存放到themes目录下。随后返回博客目录，修改_config.yml文件。


# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: landscape        ##修改成你下载的主题的名字，这里landscape是Hexo默认的主题名字
# Deployment

保存配置文件之后，三连：hexo clean、hexo g、hexo s来查看是否生效。随后hexo d推到github。

最后只需要购买域名并在后台把域名解析到你的github地址即可。

另外搭建的博客选用的主题可能并不完全符合本身期望 ，故要对其进行修改，如添加about等独立页面，只需要使用命令hexo new page <your_page_name>即可。

美化主题#

添加萌萌哒#

首先先安装hex-helper-lice2d插件.


g7s:~ $ npm install --save hexo-helper-live2d

随后将以下代码添加到主题配置文件_config.yml即可。


live2d:
  enable: true
  scriptFrom: local
  pluginRootPath: live2dw/
  pluginJsPath: lib/
  pluginModelPath: assets/
  tagMode: false
  log: false
  model:
    use: live2d-widget-model-shizuku
  display:
    position: right
    width: 150
    height: 300
  mobile:
    show: true

当然Live 2D不单只有shizuku，还有其他模型，可以自行百度添加哦。

解决问题#

执行hexo d操作后需要输用户名和密码#

首先需要设置SSH。


g7s:~ $ cd ~/.ssh 

g7s:.ssh $ sudo ssh-keygen -t rsa -C "你的邮箱"   ##生成SSH，随后三次回车

g7s:.ssh $ sudo cat /var/root/.ssh/id_rsa.pub  #打开id_rsa.pub文件，并把内容复制到剪切板，可能需要安装gedit工具

随后登陆到github，进入设置 –> SSH and GPG keys，点New SSH。

提交后确认一下账号密码。紧接着需要配置SSH，步骤如下：


g7s:.ssh $ cd ~/.ssh
g7s:.ssh $ sudo touch config             
g7s:.ssh $ sudo vim config
g7s:.ssh $ sudo chmod 600 config

config文件的内容为：


Host github.com  
User 你的注册邮箱 
Hostname ssh.github.com  
PreferredAuthentications publickey  
IdentityFile ~/.ssh/id_rsa  
Port 443

测试一下，发现成功连接。


g7s:.ssh $ sudo ssh git@github.com
PTY allocation request failed on channel 0
Hi iamg7s! You've successfully authenticated, but GitHub does not provide shell access.
Connection to github.com closed.

最后只需要修改根_config.yml文件，将deploy下的repo属性修改成下面形式，保存即可。


deploy:
  type: git
  repo: git@github.com:你的name/你的username.github.io.git
  branch: master
